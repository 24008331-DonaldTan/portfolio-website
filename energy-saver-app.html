Energy Saver Monitor

Client-server mobile application for energy usage monitoring.

Overview

It is a client-server mobile application that allows users to monitor household appliance energy usage and estimate electricity costs.

Conceptualisation

The idea was to support sustainable living by helping users better understand how much energy their household appliances consume. The application connects a mobile frontend to a backend web service for persistent data storage.

Planning

  • Designed a client-server architecture
  • Planned RESTful API endpoints for CRUD operations
  • Defined an appliance data model (name, power, hours, cost)
  • Designed a simple and intuitive mobile UI

Development & Implementation

The mobile application was built using React Native with Expo, while the backend web service was implemented using Node.js and Express. Appliance data is stored in a SQLite database and accessed via RESTful API calls.

Key Features

  • Add, edit and delete appliances
  • Automatic energy and cost calculation
  • Data retrieved from backend web service
  • Persistent storage using SQLite
  • Clean and user-friendly mobile interface

Challenges & Solutions

One challenge was ensuring accurate energy and cost calculations while keeping the UI responsive. This was addressed by performing calculations on the backend and returning computed values via the API.

Reflection

This project strengthened my understanding of client-server architecture, RESTful API integration and collaborative development. It also improved my ability to integrate frontend and backend components effectively.

Tools & Technologies

React Native Expo Node.js Express SQLite REST API
