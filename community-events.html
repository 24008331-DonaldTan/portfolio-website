Community Events Web Application

Client–server web application for managing and browsing community events.

Overview

This project is a community events web application developed as part of a school assignment. The application allows users to create, view, edit and delete community events through a simple and intuitive interface.

The project follows a client–server architecture, where the frontend handles user interaction and the backend manages data storage and retrieval.

System Architecture

  • Frontend: Built with React for UI rendering and user interaction.
  • Backend: Node.js and Express providing RESTful APIs.
  • Database: SQLite used to store event data persistently.
  • Communication: Frontend communicates with backend using HTTP requests.

Key Features

  • Create new community events using a form-based interface
  • View a list of existing events retrieved from the database
  • Edit event details through backend API calls
  • Delete events with immediate UI updates
  • Persistent data storage using SQLite

Development & Implementation

The frontend application is built using React and communicates with a Node.js backend via RESTful API endpoints. Each CRUD operation (Create, Read, Update, Delete) is handled by the backend, ensuring separation of concerns and clean system design.

During development, the backend was run locally and tested using browser and frontend API requests to ensure correct data flow between the client and server.

Reflection & Learning

This project strengthened my understanding of full-stack web development, particularly how frontend and backend systems communicate. I learned how to design RESTful APIs, manage persistent data, and structure a project using a client–server architecture.

In the future, the application could be enhanced with user authentication, improved validation and deployment to a cloud hosting platform.

Tools & Technologies

React Node.js Express SQLite
